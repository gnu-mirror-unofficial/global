Mailing lists

There are several mailing lists associated with GNU GLOBAL.

Bug report Mailing list

If you think you have found a bug or a suggestion for improvements in GLOBAL, then please send a mail to <bug-global@gnu.org>.

This list distributes, to the active maintainers of GLOBAL, bug reports and fixes for, and suggestions for improvements in GLOBAL. This list is also for user discussion.

At least, you should describe the followings about the bugs:

  • What did you do? (command line)
  • What did you expect from it?
  • What was occured? (as is)
To subscribe this list, please visit http://mail.gnu.org/mailman/listinfo/bug-global.
See also General Information about bug-* lists and reporting program bugs.
To see the collection of prior postings to the list, visit the Bug-global Archives.

Help Mailing list

If you need help about GLOBAL, then please send a mail to <help-global@gnu.org>.

This list is the place for authors, users and installers of GLOBAL to ask for help.

Please obey the following manners:

  • Before sending a mail, please read 'FAQ' and tutorial(doc/global.txi) in the GLOBAL package.
  • Don't send bug reports or suggestions to this list; mail them to bug-global@gnu.org instead.
  • Don't send private mails to the person who answered your questions.
  • Don't send HTML mail.
  • Please describe the problem CONCRETELY.
To subscribe this list, please visit http://mail.gnu.org/mailman/listinfo/help-global.
See also General Information about help-* lists.
To see the collection of prior postings to the list, visit the Help-global Archives.

Announcements Mailing list

This list distributes announcements and progress reports on GLOBAL.

To subscribe this list, please visit http://mail.gnu.org/mailman/listinfo/info-global.
See also General Information about info-* lists.
To see the collection of prior postings to the list, visit the Info-global Archives.
